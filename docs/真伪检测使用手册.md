# Claude 真伪检测 · 使用手册

> 目标：你（小小）不依赖 Atlas，自己就能对一个网关/中转站跑完整真伪检测，判断它是不是真的在提供官方 Claude。

---

## 它能检测什么

四类"假冒官方 Claude"的攻击：

| 攻击 | 含义 | 靠什么探针 |
|---|---|---|
| 换模型 / 套壳 | 后面其实是 GPT 等，套个 Claude 接口壳 | 协议指纹 / SSE / 错误体 |
| 换 tokenizer | 用了非 Claude 的分词器 | token 差分 |
| 假 1M 上下文 | 声称支持长上下文，实际静默截断 | needle 召回 |
| 偷降级 | 还是 Claude，但偷偷换成更小的（opus→haiku） | 能力锚点通过率 |

最终给四类结论之一：**✅真·官方 / ⚠️疑似降级 / ❌疑似套壳 / ❔证据不足**，附分级证据链。

---

## 一次性准备（只做一次）

### 1. 填要检测的网关

编辑 `configs\providers.local.json` 里的 `suspect_model`（已有这个角色，改成你要测的网关）：

```json
"suspect_model": {
  "provider_id": "suspect_xxx",
  "base_url": "https://要测的网关地址",
  "model": "claude-opus-4-6",          // 它声称提供的模型
  "api_key_env": "SUSPECT_MODEL_API_KEY",  // 不要直接写 key！写环境变量名
  "protocol": "anthropic_messages",     // Claude 原生用这个；若它走 OpenAI 格式用 openai_chat
  "auth_type": "x-api-key"              // Anthropic 原生用 x-api-key；OpenAI 格式用 bearer
}
```

### 2. 填 key（key 永远不进配置文件，只进 secrets）

编辑 `local_secrets.env`，把对应的 key 填到等号后面：

```
SUSPECT_MODEL_API_KEY=你的key
```

> ⚠️ `local_secrets.env` 已被 gitignore，不会进版本库。**绝不要把 key 写进 providers.local.json**。

### 3. 确认基线就绪

检测需要一个【可信的真官方 Claude】当对照基线。默认基线 `OFFICIAL-CLAUDE-OPUS46` 已经建好（drhknode 真官方 opus-4-6 的指纹 + 能力通过率 1.0）。

确认存在：`baselines\OFFICIAL-CLAUDE-OPUS46\baseline.json` 在即可。

> 如果要换别的可信源当基线，先把 `tested_model` 指向那个真官方源，再跑：
> ```powershell
> python .\eval_cli.py baseline --provider tested_model --baseline-id 我的基线名 --live --request-delay 1.0
> python .\eval_cli.py capability-probe --provider tested_model --baseline-id 我的基线名 --live --request-delay 1.0
> ```

## 跑检测

### 第一步：dry-run 演练（不花额度，先验证配置对不对）

```powershell
.\scripts\run_authenticity_check.ps1
```

看到 `[OK] 基线就绪` + 命令正常组装 = 配置没问题。dry-run 结论一定是"证据不足"（因为没发真实请求），这是正常的。

### 第二步：真实检测（调网关、花额度）

```powershell
.\scripts\run_authenticity_check.ps1 -Live
```

它会自动跑：协议指纹 + SSE + 错误体 + 能力锚点四类探针，输出中文判定。

### 可选：加跑假 1M 探针（慢、贵，>200K 的大请求）

```powershell
.\scripts\run_authenticity_check.ps1 -Live -Full
```

## 怎么读结果

脚本最后会打印一段中文报告：

```
结论: ✅ 真·官方   （或 ⚠️疑似降级 / ❌疑似套壳 / ❔证据不足）
置信度: 0.78
强证据（定罪级）:
  · stop_reason_enum: baseline=... | observed=...
  · capability_anchor: baseline=1.0 | observed=...
佐证（参考，单独不定罪）:
  · tokenizer_delta: ...
仅参考（样本不足/不计票）:
  · ...
```

读法：
- **结论**先看这个。
- **强证据**是定罪级——比如 stop_reason 是 OpenAI 方言、能力通过率远低于基线，单条就能决定方向。
- **佐证**单独不足为凭，叠加提升置信。
- **仅参考**是样本不足/网关行为差异，不计票。
- 看到 **❔证据不足** + 理由含 `429`/`failure`：不是判它假，是**没测成**（限流/配额/key 失效），换健康 key 重试。

## 常见卡点

| 现象 | 原因 / 解法 |
|---|---|
| `找不到基线` | 基线没建，按"一次性准备 3"建，或用默认 OFFICIAL-CLAUDE-OPUS46 |
| 结论一直"证据不足" + 大量 429 | 网关限流/配额。加大 `-Delay 2.0`，或换健康 key |
| 脚本中文乱码/解析报错 | 脚本需 UTF-8 BOM（已处理）。若再编辑过，确保存成 UTF-8 |
| Claude 走 OpenAI 格式报错 | suspect 的 `protocol` 改 `openai_chat`、`auth_type` 改 `bearer` |
| 想换检测目标 | 只改 `providers.local.json` 的 `suspect_model` + `local_secrets.env` 的 key |

---

## 一句话总结

1. 填 `providers.local.json` 的 `suspect_model` + `local_secrets.env` 的 key
2. `.\scripts\run_authenticity_check.ps1`（先 dry-run）
3. `.\scripts\run_authenticity_check.ps1 -Live`（真实检测）
4. 看中文结论 + 强证据

> 黑盒限制：本工具能证明"这个接口在这些证据下表现得像/不像官方 Claude"，能发现套壳/降级/假长上下文，但不能单独证明背后一定是官方上游——真正的身份保证仍需官方账号/合同/日志。
