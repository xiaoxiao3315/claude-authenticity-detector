# 如何使用本工具（网关真伪 + 性能评测）

本工具用于验证一个第三方 API 网关/中继：**它提供的是不是真官方模型、性能能否达到宣称的配额**。
下面这份说明既可自己照着跑，也可以**整段复制转发给别人的 AI 助手**（Claude / GPT / Cursor 等），让它驱动命令行完成评测。

---

## 给 AI 助手的操作指令（可整段转发）

我有一个网关真伪 + 性能评测工具（Python），已解压在本地。请按下面步骤运行，遇到报错就读源码排查。

### 1. 环境准备
- 需要 Python 3.11+
- 装依赖（只依赖 httpx）：
  ```
  pip install -r requirements.txt
  ```
- 跑离线自检确认能用（不联网、不花钱）：
  ```
  python bench_access_report.py --self-test
  python bench_gateway.py --self-test
  ```
  两个都输出 PASS / ok 即环境正常。

### 2. 配置要测的端点
复制 `configs/providers.example.json` 为 `configs/providers.local.json`，填入要测的网关：
- `base_url`：网关地址
- `model`：模型 id
- `protocol`：OpenAI 兼容填 `openai_chat`，Anthropic/Claude 填 `anthropic_messages`
- `auth_type`：`bearer`（OpenAI 式）或 `x-api-key`（Anthropic 式）
- `api_key_env`：存 key 的**环境变量名**（不要把 key 明文写进文件）

把 API key 设成上面那个环境变量。PowerShell：
```
$env:变量名="sk-..."
```

### 3. 先探路（几分钱，确认通不通）
```
python bench_gateway.py --config configs/providers.local.json --key tested_model --rate 2 --max-requests 2 --input-len 200 --output-len 50 --max-cost-usd 1
```
看到 2/2 成功、返回正确 model，说明端点通了。

### 4. 跑正式评测（对标准入报告格式）
```
python bench_access_report.py --provider tested_model --tiers small --samples 10 --concurrency 10 --price-in 5 --price-out 25 --max-cost-usd 3
```
输出 Grade / TTFT P50-P90 / OTPS / RPM，以及真伪 VETO 标记。结果存到 `benchmarks/`。
- 复现"50 并发"档：改 `--concurrency 50 --samples 50`，`--max-cost-usd` 相应调高。
- **务必带 `--max-cost-usd`**，否则可能烧掉大量额度。

---

## 输出怎么读

- **Grade / V_TOTAL**：⚠️ 综合分是**非官方复现公式**，仅本工具内可横向比较，别拿去对某平台的绝对分。**以单项 pass/fail 和 VETO 标记为准。**
- **TTFT P50/P90**：首 token 延迟；**OTPS**：输出速度 tok/s；**RPM/TPM**：实测吞吐 vs 宣称配额。
- **VETO 标记**（真伪判定，最有价值）：
  - `server-model 不匹配`：返回的模型和请求的不一致 → 套壳
  - `尺寸依赖退化`：小请求过、大请求崩 → 供给不足的逆向资源
  - `resp_ on openai-chat`：response-id 是 `resp_` 前缀 → ChatGPT/Codex 账号池，不是标准 OpenAI API（应为 `chatcmpl-`）

## 注意事项

- 用**一次性 / 可弃 key**。压测会真实消耗额度，且高并发可能触发网关封禁。
- **失败本身就是结论**：大 payload / 高并发下大量 503、RemoteProtocolError、504，说明网关扛不住宣称负载，不是工具 bug。
- 无 `x-ratelimit-*` 响应头 = 客户端无法预知限额，也是一条真伪信号。

## 进阶

`eval_cli.py` 提供更完整的真伪判定（`quickcheck` 子命令、指纹基线等），偏专业。**日常评测用 `bench_access_report.py` 一个就够**，输出格式最接近正式准入报告。
